Context

Solargraf needed a cohesive icon language

Solargraf is an end-to-end solar design software that streamlines everything from design and proposals to permitting for solar installers. The product had grown rapidly, and its iconography reflected that — icons were sourced from multiple places, had inconsistent stroke weights, mixed metaphors, and no unifying grid or style guide.

As the sole designer on this initiative, I took ownership of rebuilding the entire icon system from scratch — establishing a visual grammar that could scale with the product while maintaining clarity at every size.

What's the Need?

Problems in the current icon design

An audit of the existing icon system surfaced problems that went beyond aesthetics:

🔀
Irrelevant icons for various functionalities
Icons didn't communicate their intended meaning. For example, the Electrical Markers icon looked like a plug instead of representing markers on the design canvas.
⚖️
Inconsistent icon design language
Combination of filled and outline icons used inconsistently. Different visual weights — the same icon reused for Project and View Company.
🆕
New icons needed for new features
As features grew, the product required new icons that couldn't be sourced from existing mismatched libraries while maintaining visual coherence.
🧩
Icons sourced from different places
Icons pulled from multiple libraries with varying styles, weights, and visual logic — no single source of truth existed.
Key insight: The inconsistency wasn't just visual — it was cognitive. Users were learning different visual languages in different parts of the same product, creating unnecessary cognitive load.
Before — Issues found
✕Mixed stroke weights (1px, 1.5px, 2px) within the same view
✕Inconsistent corner radii — sharp corners next to fully rounded icons
✕No optical size compensation — icons appeared different sizes
✕Same icon used for Project and View Company
✕Icons with different visual weights and out of context
After — Goals set
✓Unified 1px stroke weight across the entire set
✓No corner radius except when shape demands it
✓24px base grid with 18px optical safe zone
✓One metaphor per concept — clear, unambiguous meaning
✓Rounded stroke caps & joins for a cohesive feel
Design Checklist

The rules behind every pixel

Before drawing a single icon, I defined a strict checklist that governed every design decision. This ensured consistency at scale and made the system teachable to other designers.

Grid Size24 x 24 px
Optical Size18 x 18 px
Stroke Width1 px
Element SizeIn whole numbers, preferably even numbers
Stroke AlignmentCentrally aligned with the grid line
Gap b/w ElementsMinimum 1 px gap between elements
Shape Cut1 px gap around elements
Corner RadiusNo corner radius except if needed to define the shape
Corner Join, Stroke CapRounded
Additional ElementsBottom right of the main element
Process

From audit to Figma component library

The project followed a clear four-phase approach, from cataloguing the existing mess to shipping a production-ready component library.

Phase 01
Understanding & Audit
Analysed Solargraf's existing icon set to identify inconsistencies and usability issues. Gathered understanding on how users interact with icons and the context of use.
Phase 02
Ideation & Exploration
Explored different variations for line icons to analyse elements visually. Explored filled versions, simplified where needed, ensured consistent proportions.
Phase 03
Refinement & Testing
Tested icons at minimum and maximum sizes to ensure they communicate well. Ensured breathing space and avoided weird shape cuts.
Phase 04
Library & Handoff
Built a Figma component library with search-friendly naming, size variants, and usage documentation. Exported optimized SVGs for engineering.
The Icon Set

A glimpse of the redesigned icon set

40+ icons organized across 4 product areas. Each icon designed on a 24px grid with 18px optical safe zone, 1px stroke weight, and rounded joins — in both outline and filled variants.

Impact

A system, not just a set of icons

The redesigned icon system didn't just look better — it fundamentally changed how the team worked with icons. The consistent visual language improved the perceived quality of the entire product.
