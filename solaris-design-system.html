The Need

The old system was slowing everyone down

🔓
No token system
Hard-coded colours, spacings, radii. Every global change meant hundreds of manual updates.
✂️
Constant detaching
No auto-layout, fixed widths. Designers detached components every single time.
🧩
Missing components
New UI patterns never added back to the library — growing drift between system and product.
📐
Inconsistency everywhere
12+ greys, varied spacings, inconsistent radii — the product felt like several different apps.
My Contribution · Audit

Finding where things were breaking

🩶
12+ shades of grey
No naming convention — every surface used a slightly different grey.
↔️
Inconsistent spacing
12px here, 14px there, 16px somewhere else. No scale defined.
⬜
Radius mismatch
8px on buttons, 10px on cards, 12px on modals — none intentional.
🔤
Typography drift
Headings, labels, and body copy had no shared type scale.
Token Architecture

Two-tier token system

Raw tokens hold primitive values (grey-500, spacing-8). Semantic tokens reference them with meaning (text-secondary, surface-mainAction). This separation lets us update contexts independently and enables theme switching without touching components.
